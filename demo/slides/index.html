Where this came from

Claude Code made agent delegation feel real.

1 Plan Start with a real decision, not a vague "analyze this" prompt.
2 Fan out Send focused subagents at the same context from different angles.
3 Synthesize Collapse agreement and disagreement into something a human can act on.

Agent Swarm is my attempt to make that pattern explicit, inspectable, and reusable across harnesses.

The core loop

One command creates a decision panel.

agent-swarm run 1 "<decision question>" --preset <panel> --doc <context> --quiet
Preset Which roles are in the room, and what choices exist.
Agents Role, prompt, harness, and model are explicit files.
Artifacts Every run leaves agent output, manifest, events, synthesis.
Decision The output is a recommendation plus the argument around it.
Not "ask the model better" Make the workflow observable

Closeout

The point is not more agents. It is better decisions.

1
Make the room explicit Roles turn fuzzy prompting into a repeatable decision surface.
2
Keep the run inspectable Agent outputs, manifest, events, and synthesis survive the demo.
3
Swap the infrastructure Harnesses and models can change without renaming the roles.
4
End with action The best synthesis becomes a next step, a brief, or a PR.

Try it with one real decision, three roles, two docs, and one round. Then inspect what the disagreement tells you.
