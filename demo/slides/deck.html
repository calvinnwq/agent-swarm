01 · whoami

I build tools for my agents.

I'm Calvin. I build tools that make my agents better at building the things I actually want. Agent Swarm came out of a simple frustration: I kept asking one model a fuzzy question when what I really wanted was a few useful opinions and a decision.

agent-swarm · meetup demo github.com/calvinnwq/agent-swarm
02 · why it exists

Subagents made the pattern click.

Watching Claude plan work, hand pieces to focused subagents, and pull the answers back together made something obvious: a lot of the magic is in who you put in the room. Agent Swarm turns that into a small CLI workflow I can inspect, rerun, and tweak.

$ agent-swarm run 1 "<decision>" --preset <panel> --doc <context> --resolve off --quiet
less prompt whispering more explicit workflow
03 · the shapes

Five panels I keep reaching for.

Expert Panel Product, engineering, and design sanity-check the next move. Live today
Adversarial Review One agent sells the idea, one pokes holes, one finds the safe slice. Live today
Customer Panel Fake users, real friction. Useful for first-run clarity. Live today
Audit Lanes Docs, DX, release, and safety checks without one giant reviewer. In the kit
Build Triage Turns scope, risk, and tests into a brief someone can actually build. In the kit

07 · the takeaway

The win is a decision I can reuse.

01

A repeatable room

The same panel can look at the next question tomorrow.

02

Named opinions

Each voice has a job, a file, and a point of view.

03

Evidence on disk

I can read the agent outputs, manifest, events, and synthesis.

04

Swap the engine

Codex, Claude, OpenCode, whatever. The role stays the role.

The useful bit is the shape: a real question, a few opinionated roles, and a synthesis that can become a next step, a brief, or a PR.
