03 —Concepts

What is a swarm?

A swarm is a panel of AI agents — each playing a different role — that work the same problem at the same time and hand you back a single, deterministic answer. No deliberation, no coin-flip. The synthesis is computed from what the agents said, not decided by another model.

The shape of every run

Every swarm follows the same four steps, regardless of the preset or the question you ask:

Every swarm: pick agents → run rounds → resolve → one synthesis

Pick agents. A swarm has between 2 and 5 agents. Each agent has a name, a persona, and a prompt that shapes how it reads the question. You can use a bundled preset — a curated roster with a decision vocabulary already chosen — or assemble your own agents from YAML or Markdown files.

Run rounds. Every agent in the panel runs in parallel. A four-agent panel takes about as long as a single agent because they all run at the same time. You can run 1 to 3 rounds; each additional round gives agents a chance to react to what the prior round surfaced.

Resolve. Between rounds, the swarm decides what to carry into the next brief. With --resolve off, a deterministic template summarises the prior packet — no extra model call. With --resolve orchestrator, a bundled orchestrator agent reads the prior packet and produces a structured directive for the next round.

One synthesis. Once all rounds are done, the swarm computes a synthesis. This step is fully deterministic — no model decides the final word. The same agent notes will always produce the same synthesis: consensus stance, top recommendation by confidence, shared risks, deferred questions, and an overall confidence level.

What you get back

The synthesis is the whole point. It folds the panel into one recommendation in the preset's own decision vocabulary — words you chose, not words a model invented. Here is what a customer-panel synthesis looks like after two rounds:

Synthesiscustomer-panel · round 2 · 3 agents

Fix now — onboarding hides the one action that shows value.

consensus2 of 3 agree
confidencehigh
Still openDoes the empty state need a sample project, or just a clearer first step?

A few things to notice:

  • Verdict word. Fix now is a word from the customer-panel preset's decision vocabulary — it always comes from the preset, never from a model making up language at synthesis time.
  • Consensus. 2 of 3 agents agreed on the same stance. The third held a different view; that is surfaced, not hidden.
  • Confidence as a level. Confidence is reported as low, medium, or high — never a fabricated percentage. It is the rounded average of every agent's self-reported confidence across all rounds.
  • Still open. Questions the panel could not resolve are carried through so you know exactly where the uncertainty lives.
Note

The synthesis file lands at .agent-swarm/runs/<timestamp>-<slug>/synthesis.md. The full run directory also contains per-agent round outputs, the event ledger, and a durable checkpoint.

Run one in a line

No project config required. Pass the number of rounds as the first positional argument (1–3), then your question, then an optional preset:

# install once
npm install -g @calvinnwq/agent-swarm

# run a two-round customer panel (2 = rounds)
agent-swarm run 2 "Should we ship the freemium tier this quarter?" \
  --preset customer-panel

The 2 before the question is a required positional argument — the number of rounds — not a flag. Round count is always explicit on the command line, from 1 to 3.

While the swarm runs you will see a live panel in your terminal: one row per agent, elapsed timers, and a phase banner that moves through rounds. When it finishes, the synthesis prints and the full run directory is written to .agent-swarm/runs/.

Note

Real harnesses can take longer than the default 120 s per-agent timeout. Add --timeout-ms 300000 for deeper questions. Use --quiet for one-line-per-event output in CI.
