Run a panel of AI agents. Get one deterministic answer.

Agent Swarm is a TypeScript CLI that fans out 2–5 agents over 1–3 rounds, validates their structured JSON output, and produces a single deterministic synthesis you can review or check in.

Get started → Command reference

Why a swarm

One model answering alone is a single point of view. Agent Swarm runs a small panel — a product manager and a principal engineer, an adversarial review pair, a customer panel — in parallel, then folds their answers into a deterministic synthesis: consensus, the top recommendation by confidence, shared risks, and the questions still open. No LLM call decides the final report, so the same packets always produce the same synthesis.

How it works

Every agent-swarm run follows the same lifecycle:

  1. Plan. CLI flags, project config, and the chosen preset are merged. Each selected agent resolves a runtime harness.
  2. Round. Agents run in parallel (concurrency 3 by default) and return JSON validated against the agent output schema.
  3. Between rounds. Depending on --resolve, the swarm builds the next directive deterministically or runs an LLM orchestrator pass.
  4. Synthesize. Once the last round finishes, a deterministic synthesis is produced — no LLM call — with consensus, top recommendation, shared risks, and deferred questions.

The README is the authoritative user-facing contract; this site mirrors it for quick reference. For the durable spec see SPEC.md.
