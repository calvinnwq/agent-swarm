06 —Operations

What ships, and how

Every release clears the same deterministic verification gates, then Release Please cuts the tag and GitHub Release from Conventional Commits.

Verification gates

These run in CI on every pull request and must be green together before anything merges to main:

Gate Command
Formatting pnpm format:check
Lint pnpm lint
Typecheck pnpm typecheck
Unit tests pnpm test
Build pnpm build
Smoke pnpm smoke

Release process

Releases are managed by Release Please from Conventional Commits on main. Release-driving commit types such as feat:, fix:, and deps: appear in the next release; docs:, test:, refactor:, and chore: do not.

Merging the Release Please PR updates package.json, writes CHANGELOG.md, creates a git tag, and creates the GitHub Release. Tags use plain vX.Y.Z format. npm publishing is not part of the current release workflow.

Note

The full readiness checklist lives in docs/release-readiness.md, and the operator runbook in docs/release-operations.md. Do not publish, retag, or edit releases without explicit approval.
